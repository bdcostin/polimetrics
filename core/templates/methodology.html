Data Weighting

In order to improve the accuracy of our results and take in to consideration a few of the many varaibles, we have weighted our data using the formula below.

Engagement = number of retweets + number of likes in each tweet. The engagement values are how we measure a tweet's influence on Twitter; the larger the engagement the greater the role it plays in shaping political discourse. These engagement levels are used in the formula for weighting the data. A tweet's engagement divided by the total engagement, i.e. the sum all individual tweet engagements with respect to each candidate, is the proportion we use to multiply each sentiment evaluation to. This product is summated to achieve the daily and overall trend lines seen in each candidate's detail line graph. This is a concept known as weighted averaging, i.e. the method of computing an arithmetic mean of a set where some elements carry more importance (weight) than others.

With this in mind, engagement helps us derive proportionality through a tweet's activity. Here we achieve two evaluation types: Aggregate averages--computed by adding all the weighted averages together--and actual daily averages--where the proportion accounts only for the engagement total for that day, ignoring the engagement totals in previous days.

We begin our process by collecting Tweets with Twitter's Standard search API. This allows us to specify, and therefore return Tweets about, each candidate. Results can be returned with various search parameters such as candidate's first and last name, last name, hastags, and mentions. Unfortunately the Standard search API does not return all Tweets that are about the searched candidate, instead the results are a sampling of random but relevant Tweets. Given that the relevance is at play we have assumed that results received are a truly random sampling

As we collect Tweets, we use a regular expression to remove all parts that are not useful for calculating sentiment, i.e., links, emojis, and @ symbols. This step is essential to ensure that the evaluation of sentiment is not misguided by information that does not have explicit meaning.

We then use TextBlob (an API for natural language processing) to evaluate the cleaned Tweets' sentiment. TextBlob uses two sentiment analysis implementations, PatternAnalyzer and NaiveBayesAnalyzer. The PatternAnalyzer is based on the pattern library. While the NaiveBayesAnalyzer is an Natural Language Toolkit classifier trained on a corpus of movie reviews.

TextBlob returns two values when performing sentiment anaylsis on a text, polarity and subjectivity. Polarity uses the wording of a text to determine whether it is positive or negative and returns a float between -1 and 1, with -1 being completely negative and 1 being completely positive. Subjectivity refers to how direct, and therfore how explicit, the text is, which also provides a metric for the accuracy of the polarity. The value returned is a float ranging from 0 (very objective) to 1 (very subjective).

Social media is home to many political communities segregated by partisan lines. The mission of polimetrics.io does not debunk the insulur nature of the politically active Twitter users. [1,2]. Polimetrics.io aims to provide visual representations of the current political landscape. We include all tweets despite any political biases that might influence our data sample.
